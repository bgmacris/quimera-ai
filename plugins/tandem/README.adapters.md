# tandem — Cursor & OpenCode adapters

Multi-IDE adapters for [tandem](https://github.com/bgmacris/quimera-ai/tree/main/plugins/tandem): shared Chrome (human + agent, CDP + Playwright MCP) with per-site navigation memory (`tandem:map`).

Upstream core targets **Claude Code** via the quimera-ai marketplace. This fork adds thin adapters for **Cursor** and **OpenCode** without modifying upstream scripts (see [UPSTREAM.md](./UPSTREAM.md)).

## Requirements

- Google Chrome or Chromium (autodetected on macOS and Linux)
- Node.js 22+ (`npx` for Playwright MCP)
- **Linux** or **macOS** (Windows: use WSL2 as Linux)

## Quick install

From this directory (`plugins/tandem/`):

```bash
chmod +x install.sh adapters/shared/mcp-wrap.sh adapters/cursor/scripts/*.sh
./install.sh
```

Auto-detects Cursor and OpenCode; always installs CLI wrappers in `~/.local/bin/`.

Flags: `--all`, `--cursor`, `--opencode`, `--dry-run`.

Ensure `~/.local/bin` is in your `PATH`.

## Layout

```
plugins/tandem/
├── scripts/, bin/, skills/, …     ← upstream core (do not edit)
├── adapters/
│   ├── shared/                    ← mcp-wrap, env template
│   ├── cursor/                    ← .cursor-plugin, hooks, mcp.json
│   └── opencode/                  ← tandem-map-inject plugin, config fragment
├── install.sh
└── UPSTREAM.md
```

## Data directory

Default: `~/.local/share/tandem` (Chrome profile, site profiles, output).

Override: `TANDEM_DATA_DIR` in `~/.local/bin/tandem-env.sh`.

Site profiles: `~/.local/share/tandem/sites/<host>.md`

## Cursor

After `./install.sh`:

1. **Developer: Reload Window**
2. `/tandem:browser-start` · `/tandem:browser-status` · `/tandem:browser-stop`
3. Skills `/tandem` and `/map`; agent `web-navigator`

Plugin symlink: `~/.cursor/plugins/local/tandem` → `adapters/cursor/`

## OpenCode

After `./install.sh`:

1. Restart OpenCode
2. `/browser-start` · `/browser-status` · `/browser-stop`
3. MCP server `tandem`; plugin auto-injects `tandem:map` on `browser_navigate`

Config merge: `adapters/opencode/merge-opencode-fragment.mjs` patches `~/.config/opencode/opencode.jsonc`.

Project-level commands (e.g. `/smoke`, `/verify`) stay in each repo's `.opencode/commands/`.

## Claude Code

Use upstream marketplace (no adapter symlink):

```
/plugin marketplace add bgmacris/quimera-ai
/plugin install tandem@quimera-ai
```

## Environment

Generated by install: `~/.local/bin/tandem-env.sh`

| Variable | Purpose |
|----------|---------|
| `TANDEM_PLUGIN_ROOT` | Path to this plugin directory |
| `TANDEM_DATA_DIR` | Persistent Chrome + site data |
| `TANDEM_CHROME_BIN` | Optional Chrome override (WSL edge cases) |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `ECONNREFUSED 127.0.0.1:9222` | `tandem-browser start` or `/browser-start` |
| First browser call fails after restart | Retry once (MCP CDP reconnect) |
| Chrome not found | Set `TANDEM_CHROME_BIN` in `tandem-env.sh`, re-run install |
| Cursor: `tandem:map` profile never appears in context | Known Cursor limitation — see below. OpenCode and Claude Code are unaffected. |

## Known limitations

**Cursor — `tandem:map` auto-injection may not reach the model.** The Cursor adapter
injects the site profile via a `postToolUse` hook returning `additional_context`, which
is the field Cursor documents for that hook. As of Cursor 2.x there is an open bug where
`postToolUse` hooks accept and log `additional_context` but the injected context is *not*
surfaced to the model ([forum report](https://forum.cursor.com/t/native-posttooluse-hooks-accept-and-log-additional-context-successfully-but-the-injected-context-is-not-surfaced-to-the-model/155689)).
Until Cursor fixes this, invoke the `/map` skill manually to load a site profile in Cursor.
The hook code is correct against the [Hooks reference](https://cursor.com/docs/hooks) and
starts working once the platform bug is resolved. OpenCode injects directly into the tool
output and Claude Code uses `hookSpecificOutput.additionalContext`, so neither is affected.
